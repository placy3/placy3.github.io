小白尝试搭博客中

Hugo建博客时遇到的一些问题：

不加载css，显示不安全

因为github给页面用的是https://，所以确保配置文件里baseUrl写的也是https://，html中的css/js引用链接（自己的库）就也是“安全”的了。

config里设定了tag = “tags”，文章开头也写了tags= [“xx”, “xxx”]，但是博文的tag死活不变

简单的解决方案：换主题。
详细的解决方案：检查主题的layouts/_default/文件夹（网站根目录的layout默认是空的），single是单篇博文，打开检查生成方式。
像

{{ if isset .Params "tags" }}
tags:
{{ range .Params.tags }}{{ . }} {{ end }}

这样的才是真正会创建tags的

localhost的404页面始终是404 page not found

用于生成的404.html是放在/layouts/下无误，但是只有放在服务器上（例如github pages啦）才会在404的时候用这个html渲染页面。原理我不清楚……
或运行hugo之后在public里面打开404.html查看即可
